LAB / podcast_mind

Spotify podcast 向けの小さな Web アプリは作れる。

ただし、3つの要件のうち Spotify ログイン と podcast 再生 は公式手段で進めやすく、 Spotify が表示している文字起こしの取得・コピー は公式 API の公開範囲次第で止まる可能性が高いです。

実装しやすい

Spotify ログイン

OAuth でログイン可能。Flask 側で callback を受ける形か、フロント側 PKCE のどちらでも組めます。

実装しやすい

podcast 再生

episode embed を使えば再生 UI は作れます。Premium 前提まで広げるなら Web Playback SDK の検討が必要です。

要確認

文字起こしコピー

Spotify の transcript を API で取れないなら、サイト内でコピー機能を提供する実装は難しいです。

現実的な構成

1. ログイン

  • Spotify App を developer dashboard で作成
  • redirect URI をこの Flask サイトに登録
  • 認証後に access token を保持してユーザー情報を取得

2. episode 再生

  • まずは Spotify Embed / iFrame API で始めるのが安全
  • 再生開始、停止、episode 切り替え程度なら十分狙える
  • 独自プレイヤー化したいなら Premium 条件を含め再検討

3. transcript

  • API で transcript 本文が返るならコピー機能を追加可能
  • 返らない場合、iframe 内 DOM を外から読む実装は前提にしない方がよい
  • 代替案は「手入力」「別ソース」「Spotify へ遷移してユーザー自身がコピー」

次の判断

  • まずは transcript を除いた 2 要件で MVP を切る
  • その後 transcript 取得の公式手段があるかを episode 単位で再確認する
  • もし transcript API が無ければ、要件自体を少し変える方が早いです
現時点の結論は、 「ログイン」と「再生」は高確率で可能、 「Spotify が既に持っている文字起こしをこのサイトでコピー」は公式公開 API 次第で不可の可能性あり です。
